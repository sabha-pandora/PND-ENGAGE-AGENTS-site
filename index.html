Architecture Decision

Why Multi-Agent with Orchestrator?

A single monolithic agent cannot match the depth, reliability, and scalability of specialized agents working in concert.

❌
Single Agent Approach

  • One massive prompt trying to do everything — analysis, design, automation, execution, reporting
  • Context window overflows on complex stories with 50+ test cases
  • No separation of concerns — a bad test design can't be caught before automation begins
  • Sequential bottleneck — can't parallelize performance and security testing
  • One failure mode kills the entire pipeline
  • Impossible to tune — you optimize for one task and degrade another
VS

✔️
Multi-Agent + Orchestrator

  • Each agent has a focused persona — a senior QA analyst thinks differently than an SDET
  • Structured handoffs via JSON schemas ensure data integrity between phases
  • Human review gates between agents — catch issues before they cascade
  • Parallel execution — Performance and Security engineers run simultaneously
  • Independent failure — if the Defect Reporter fails, test results are still intact
  • Each agent can be tuned, tested, and improved independently
🎯

Depth Over Breadth

Each agent is a domain expert with a focused system prompt, specialized tools, and few-shot examples tuned for its specific task.

🛡

Quality Gates

The Orchestrator enforces approval gates between agents. Bad analysis doesn't reach the designer. Bad code doesn't reach execution.

⚡

Parallel Streams

Functional and NFR analysis run in parallel. Performance and Security engineers execute simultaneously. Maximum throughput.

💰

Cost Optimization

Route complex analysis to Opus, high-volume generation to Sonnet. Each agent uses the right model for its task. No wasted tokens.

🔄

Feedback Loops

Test Monitor feeds insights back to Test Analyst. Each cycle gets smarter. A single agent has no mechanism for self-improvement across runs.

🛠

Independent Evolution

Upgrade the Security Test Engineer without touching the Test Designer. Add new agents without refactoring existing ones. True modularity.

Agent Capabilities

Skills & Capabilities

Each agent brings specialized skills to the quality engineering pipeline.

🔍 Test Analyst Skills

Requirement Parsing — Extract testable requirements from Jira REST API, Confluence docs, and Figma links
Risk Assessment — Classify by risk with failure modes, business impact, and historical defect data
Ambiguity Detection — Identify gaps with exact quotes, severity, and resolution suggestions posted to Jira
Test Strategy Context — Provide key flows, integration points, test data, and suggested test levels for Test Designer
Rich ADF Publishing — Create Confluence pages with panels, expand sections, tables, status lozenges, and layouts

✍️ Test Designer Skills

Equivalence Partitioning — Divide input domains into equivalent classes
Boundary Value Analysis — Generate tests at boundaries of valid/invalid ranges
BDD Authoring — Write Gherkin scenarios (Given/When/Then)
Test Data Generation — Produce diverse data including edge cases
State Transition Testing — Design tests for stateful workflows

📈 NFR Analyst Skills

Performance Scenario Design — Define load profiles from usage patterns
SLA Definition — Translate business requirements to measurable thresholds
Security Threat Modeling — Identify attack vectors and risks
OWASP Mapping — Map features to OWASP Top 10 categories
Architecture Analysis — Identify bottlenecks and security boundaries

⚙️ Automation Skills

Code Generation — Playwright, Cypress, Selenium, pytest, RestAssured
Page Object Model — Auto-generate POM from UI structure
API Test Generation — Build suites from OpenAPI/Swagger specs
Self-Healing Locators — Resilient selectors with fallback strategies
CI/CD Integration — GitHub Actions, Jenkins pipeline configs

⚡ Performance Engineer Skills

Script Generation — k6, JMeter, Gatling, Locust test scripts
Load Profile Design — Ramp-up, steady state, spike, soak patterns
Bottleneck Analysis — Identify perf bottlenecks from metrics
SLA Validation — Compare results against thresholds (p95, throughput, error rate)

🔒 Security Engineer Skills

OWASP Top 10 Testing — Injection, broken auth, XSS, CSRF
Auth Testing — Session management, token handling, RBAC
Data Exposure Detection — PII in responses, logs, error messages
Security Headers — CSP, CORS, HSTS, X-Frame-Options validation

How It Works

Each slash command maps to a Claude Code skill file (SKILL.md) containing the agent's full persona, instructions, REST API calls, and output schema.

Direct REST API

Agents connect to Jira & Confluence via REST API with your Atlassian API token. Fast, reliable, no MCP dependency. One-time .env setup.

Jira Ticket as Input

Pass any PVA ticket key. The agent reads the ticket, gathers context from Jira and Confluence, and produces structured output.

Implementation Plan

Build Roadmap

11 phases from foundation to full coverage. Each phase delivers standalone value.

Phase 1 — Complete
Test Analyst
Foundation agent. Reads Jira tickets via REST API, analyzes requirements, creates rich ADF Confluence pages with 6-section analysis. Adds Jira comments, labels, and approval workflow.
Phase 2 — Complete
Test Designer
Generate structured test cases tagged by level (SIT, Regression, E2E, UAT, Sanity) from Test Analyst output. BDD scenarios, test data, and edge case catalogs.
Phase 3
NFR Test Analyst
Performance + Security analysis running in parallel with functional analysis. Produces load profiles, SLA thresholds, and OWASP checklists.
Phase 4
Unit Test Generator
Shift-left quality in DEV environment. Analyzes code changes, generates unit tests, and identifies coverage gaps.
Phase 5
Test Automation Specialist
Converts designed test cases into executable automation scripts. Generates POM, API tests, and CI/CD configs.
Phase 6
Test Executor
Environment-aware execution across QA (SIT), Staging (Regression/E2E/UAT), and PROD (Sanity).
Phase 7
Regression Manager
Curates regression suite for staging. Performs change impact analysis, optimizes test selection, and prioritizes by risk.
Phase 8
Performance + Security Engineers
Parallel build. Performance engineer creates load/stress scripts. Security engineer creates OWASP/auth test suites. Both consume NFR Analyst output.
Phase 9
Defect Reporter
Environment-aware bug filing with root cause hypothesis, deduplication, and severity-based escalation across all environments.
Phase 10
Release Gatekeeper
Go/no-go decisions at three gates: QA→Staging, Staging→PROD, and PROD sanity pass/fail. Evidence-based with sign-off tracking.
Phase 11
Test Monitor
Environment-specific dashboards, cross-env quality comparison, trend analysis, anomaly detection, and feedback loop to Test Analyst.

Design Principles

Best Practices We Follow

11 core principles that guide every design decision in our agentic architecture.

01
👥

Human-in-the-Loop

Agents propose, humans approve at critical gates. Autonomy increases as trust is established. No fully autonomous decisions on production releases.

02
📄

Structured Handoffs

Agents communicate via well-defined JSON schemas, not free-form text. This ensures data integrity, parseability, and reliable downstream consumption.

03
🚀

Incremental Adoption

Start with Analyst + Designer (lowest risk, highest value). Add agents progressively. Each phase delivers standalone value before the next begins.

04
🔄

Feedback Loops

Test Monitor feeds insights back to Test Analyst. Defect patterns inform risk assessment. Each cycle gets smarter than the last.

05
🔧

Framework Agnostic

Automation agents adapt to the project's existing test framework (Playwright, Cypress, pytest) rather than imposing a new one.

06
🔗

Full Traceability

Every artifact links back to the originating requirement. Every defect links to the test that found it. Complete audit trail from story to production.

07
🛑

Fail Safe, Not Fail Silent

When an agent is uncertain, it flags for human review rather than guessing. Ambiguities are surfaced, never assumed away.

08
💰

Cost Conscious

Use the right model for the right task. Opus for complex analysis, Sonnet for high-volume generation. Token budgets per agent prevent runaway costs.

09
🌐

Environment Aware

Every agent understands which environment it's operating in and adjusts behavior accordingly. QA bugs differ from PROD escalations.

10
⬅️

Shift-Left Quality

Push quality activities earlier in the pipeline. Unit tests in DEV, test analysis at story creation. Find bugs where they're cheapest to fix.

11
🛡

NFR as First-Class Citizens

Performance and Security testing have dedicated agents and formal gates. They're not afterthoughts — they're required for staging sign-off.
